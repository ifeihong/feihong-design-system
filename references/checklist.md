# Feihong Design System · Quality Checklist
# 飞鸿品牌设计系统 · 质量检查清单

> 本清单是所有设计交付的质量门禁。每条都有具体的验证方法，不是模糊的主观判断。
> 使用方法：交付前逐条对照，P0 必须全部通过。

---

## 🔴 P0 — 必须通过（一条不过 = 不合格，必须改）

### 色彩系统

- [ ] **色值精确**：在HTML中搜索确认三种主色精确为 `#0A2463`、`#F4D35E`、`#D8315B`，没有使用近似色（如 `#0066CC`、`#FFD700`、`#E74C3C` 等通用色）
  - 验证方法：`Ctrl+F` 搜索 `#0A`、`#F4D`、`#D83` 确认色值正确
- [ ] **色彩比例可感知**：肉眼观察页面，皇家蓝是主导色调（约55%面积），金色是辅助（约30%），酒红是点缀（约15%）。不是三色平分，也不是金色/酒红抢占主色
- [ ] **金色不做大面积背景**：`#F4D35E` 和金色系色阶不用于整屏/整卡片背景，仅用于强调、装饰线、标签、小面积高亮
- [ ] **正文对比度足够**：正文文字颜色与背景对比度 ≥ 4.5:1（WCAG AA）
  - 皇家蓝 `#0A2463` 在 `#FDFBF6` 上 → 对比度 > 12:1 ✓
  - 墨黑 `#0D1225` 在 `#FDFBF6` 上 → 对比度 > 15:1 ✓
  - ❌ `#F4D35E` 金色不能在浅色背景上做正文文字
- [ ] **无第四主色**：页面中除了品牌三色+中性色+语义色（成功绿），没有出现与品牌竞争的新色相（如紫色、青色、橙色大面积使用）
  - 例外：终端绿 `#4ADE80` 仅在终端/代码场景使用
- [ ] **无纯黑纯白大面积**：背景不是 `#FFFFFF` 纯白，文字不是 `#000000` 纯黑
  - 背景应使用 `#FDFBF6`（暖米白）或深色 `#0A0E1A`
  - 文字应使用 `#0D1225`（墨黑）

### 字体系统

- [ ] **衬线+无衬线混搭**：标题（h1/h2/h3）使用 Playfair Display 衬线体，不是 Inter/Roboto/Arial 等无衬线体
  - 验证方法：检查标题元素的 `font-family` 是否包含 `'Playfair Display'`
- [ ] **非全站单一字体**：页面中至少出现两种字重/字体族的对比（衬线标题 + 无衬线正文，或 + 斜体引言，或 + 等宽代码）
- [ ] **中文正文 ≥ 16px**：中文正文字号不小于 1rem(16px)
- [ ] **全大写文字有字距**：`text-transform: uppercase` 的元素必须有 `letter-spacing: 0.08-0.15em`
- [ ] **引入了所有必要字体**：Google Fonts link 中包含 Playfair Display、Cormorant Garamond、Inter、JetBrains Mono 以及中文 Noto 字体

### 布局与结构

- [ ] **第一屏单一焦点**：Hero区域/封面页只有一个主要视觉焦点，不是多个元素争抢注意力
- [ ] **相邻section布局不同**：没有两个相邻的section使用相同的布局结构（如两个连续的居中文字section、两个连续的三列卡片网格）
- [ ] **移动端无问题**：在 ≤ 600px 宽度下，文字不重叠、不溢出容器、按钮不超出屏幕、图片不溢出
  - 验证方法：在脑中模拟或检查CSS media query
- [ ] **响应式三断点**：有桌面(>960px)、平板(600-960px)、移动端(<600px)的适配
- [ ] **clamp()流体尺寸**：关键字号（hero标题、section标题）、section间距、卡片padding使用了 `clamp()`，不是硬编码px值
- [ ] **内容宽度约束**：有 `max-width` 限制（1200px左右），在超宽屏上内容不会拉伸到全屏

### 组件与样式

- [ ] **零HTML默认样式**：
  - ❌ 没有默认 `<blockquote>` 的左边框竖线样式
  - ❌ 没有无样式的 `<ul>`/`<ol>`（必须有自定义list-style）
  - ❌ 没有默认蓝色下划线 `<a>` 链接
  - ❌ 没有默认灰色3D按钮
  - ❌ 没有默认边框的 `<input>`/`<table>`
  - 验证方法：肉眼检查所有列表、引用、链接、按钮、表格、表单
- [ ] **链接品牌化**：链接颜色为皇家蓝 `#0A2463` 或品牌色之一，不是浏览器默认蓝色；悬停有金色/品牌化效果
- [ ] **按钮层级清晰**：主按钮（皇家蓝填充）、CTA按钮（酒红填充）、次要按钮（金色/描边/ghost）有明确的视觉层级区分

### 禁忌检查

- [ ] **无Glassmorphism**：没有 `backdrop-filter: blur()` + 半透明背景的玻璃拟态效果
- [ ] **无渐变文字**：没有 `-webkit-background-clip: text` 渐变文字效果
- [ ] **无Bounce/Elastic动画**：动画使用 `ease-out` 或 `cubic-bezier(0.16,1,0.3,1)`，不是 `cubic-bezier(0.68,-0.55,0.27,1.55)` 等弹性曲线
- [ ] **无无限循环动画**：除了loading状态和极 subtle 的pulse效果，没有无限循环的动画
- [ ] **无AI光效**：没有大面积模糊光斑、发光glow效果、霓虹效果
- [ ] **无Notion/飞书式引用**：引用块不是简单的 `border-left: 3px solid` 样式，使用了金色引号+衬线斜体或其他品牌化处理
- [ ] **无多色彩虹渐变**：渐变仅限同色系深浅或径向微妙渐变，没有蓝→紫→粉等多色渐变

### 反AI检测（最高优先级）

- [ ] **反AI直觉测试**：想象这张截图发到朋友圈/推特/小红书，第一反应会不会是"又是AI做的"？如果有任何嫌疑，必须修改
  - 具体检查：是否是典型的"居中标题+三列卡片网格+CTA按钮"AI模板？
  - 是否有至少一个"人工判断"的设计决策？（非对称、元素溢出、刻意留白、不规则节奏、意外的色彩运用）
- [ ] **装饰有节制**：装饰元素（金色线、大数字、引号、圆点等）不是每页都有、不是等距出现，是少而准的
- [ ] **留白有呼吸**：不是机械等距padding/margin，有疏密对比

### 技术规范

- [ ] **HTML可直接打开**：文件保存为 `.html`，所有资源用CDN或相对路径，双击可在浏览器正常显示
- [ ] **无外部依赖缺失**：Google Fonts用CDN link，不需要本地字体文件
- [ ] **语义化标签**：使用了 `<nav>`、`<header>`、`<main>`、`<section>`、`<article>`、`<footer>` 等语义标签
- [ ] **图片有alt属性**：所有 `<img>` 有alt文本
- [ ] **DOCTYPE声明**：文件开头有 `<!DOCTYPE html>`
- [ ] **UTF-8编码**：有 `<meta charset="UTF-8">`
- [ ] **viewport meta**：有 `<meta name="viewport" content="width=device-width, initial-scale=1.0">`

## 🟡 P1 — 应该通过（尽量满足，时间允许就改）

### 品牌识别

- [ ] **皇家蓝主导**：皇家蓝承担主要品牌识别功能（导航、主标题、主按钮中均有出现）
- [ ] **酒红有呼吸感**：酒红不只是出现在一个按钮上，在合适场景有出场机会（标签、强调文字、点缀装饰），提供情绪张力
- [ ] **复古金是签名**：金色装饰线/金色圆点/金色高亮成为可记忆的视觉签名，不是无意义的装饰
- [ ] **整体气质**：清澈、艺术、专业、温暖，不是冰冷/花哨/廉价/幼稚
- [ ] **视觉惊喜**：至少有一个section打破预期（深色面板、IP形象溢出、全屏引言、极端字号对比、非对称突破等）

### 排版与节奏

- [ ] **字号极端对比**：存在超大展示文字（clamp hero标题或装饰数字）与小号说明文字（0.72-0.8rem）的强烈层级对比
- [ ] **衬线斜体引言**：Cormorant Garamond italic 至少出现一次（引言、金句、副标题），增加人文温度
- [ ] **大装饰数字/英文**：有低透明度(opacity 0.06-0.12)的装饰性大数字或英文作为视觉锚点，且不喧宾夺主
- [ ] **视觉签名贯穿**：Step 4c选的视觉签名元素（金色线/大数字/斜体引言/IP/圆点/溢出/深色面板）在全页中有呼应，不是只出现一次
- [ ] **对齐有纪律**：元素对齐有意识（左对齐/居中对齐/右对齐），不是随机放置
- [ ] **行高舒适**：标题行高1.05-1.25，正文行高1.6-1.75，肉眼阅读舒适

### 动效与交互

- [ ] **动效克制**：如有Scroll Reveal/入场动效，时长300-600ms，使用优雅缓动，不影响阅读
- [ ] **prefers-reduced-motion**：尊重用户动效偏好设置
- [ ] **焦点状态可见**：键盘Tab导航时有可见的focus样式（金色outline或皇家蓝outline）
- [ ] **触摸目标≥44px**：按钮、链接等可点击元素尺寸不小于44×44px

### 细节

- [ ] **阴影有蓝调**：阴影使用 `rgba(10,36,99,...)` 色调，不是通用黑色阴影 `rgba(0,0,0,...)`
- [ ] **圆角分级**：小圆角用于小元素（按钮4px、标签4px），大圆角用于大容器（卡片8-12px、Hero卡片20px），不是全局统一圆角
- [ ] **金色装饰线有呼吸**：标题下方金色线不是全宽，长度约为标题1/3-1/2
- [ ] **径向渐变层次**：卡片/Hero/深色面板使用了微妙径向渐变增加深度，不是纯平色
- [ ] **选中样式品牌化**：`::selection` 使用金色背景+皇家蓝文字

## 🟢 P2 — 加分项（有则更好，锦上添花）

### 艺术感

- [ ] **流动线条母题**：有细微的曲线/斜线装饰线条作为视觉母题贯穿
- [ ] **图片溢出容器**：图片/IP形象有意溢出卡片边界，制造画册感和艺术张力（不遮挡核心文字）
- [ ] **深色面板节奏**：在浅色页面中插入一个深蓝/深色面板打破节奏（深色不主导整体）
- [ ] **IP形象自然出现**：头像或character.png在合适场景出现（Hero、About、Footer），不是硬塞
- [ ] **噪点/纸质质感**：极细微噪点纹理（opacity < 3%）增加纸质质感
- [ ] **艺术面板**：有类似画册/杂志的特殊排版处理（文字叠加图片、跨栏布局、出血设计）

### 工程质量

- [ ] **CSS变量/token可复用**：颜色、字体、间距、圆角、阴影都定义在 `:root` CSS变量中，易于复用和修改
- [ ] **品牌一致性可迁移**：设计语言可以迁移到其他场景（教程→卡片→公众号）而不丢失识别
- [ ] **组件class命名清晰**：有语义化的class名，不是纯inline样式
- [ ] **打印样式**（简历/长文场景）：有 `@media print` 样式，打印时隐藏导航/动效，排版适合A4
- [ ] **Dark Mode支持**（可选）：如有深色模式，颜色转换正确，对比度足够

### 场景专属加分

不同场景有各自的加分项，见各 `scene-*.md` 文件。

## 📋 场景专属额外检查

完成通用P0/P1/P2后，根据场景类型额外检查：

### 教程型/介绍型
- [ ] 有清晰的目录/TOC导航
- [ ] 章节层级清晰（h1→h2→h3）
- [ ] 代码块有语法高亮样式（JetBrains Mono字体）
- [ ] 长文有舒适的阅读行宽（60-75ch）

### 活动页/Landing
- [ ] CTA按钮在首屏和末屏均有出现
- [ ] 有明确的价值主张（一句话说清楚做什么）
- [ ] 有社会证明/信任背书区域（如客户logo、数据、推荐语）
- [ ] 表单/转化入口清晰

### 作品集/个人主页
- [ ] 作品展示有视觉冲击力
- [ ] 项目卡片有悬停交互
- [ ] About区域有个人特色（头像/IP/金句）
- [ ] 联系方式清晰可见

### 案例研究/Case Study
- [ ] 有明确的叙事结构：背景→挑战→过程→结果
- [ ] 有过程图/草图/wireframe，不只是最终效果图
- [ ] 有数据/结果佐证
- [ ] 有反思/learnings

### 演示文稿/Deck
- [ ] 16:9比例适配（1280×720或1920×1080）
- [ ] 键盘翻页（←→/空格）
- [ ] 全屏模式适配
- [ ] 每页信息不拥挤（一页一核心观点）
- [ ] 页码/进度指示

### App型/功能型
- [ ] 有明确的功能布局（侧边栏/顶部导航/内容区）
- [ ] 交互元素有状态反馈（hover/active/disabled）
- [ ] 表单有验证样式
- [ ] 数据展示清晰（图表/卡片/列表）

### 图文卡片/小红书
- [ ] 3:4竖版比例（1080×1440）
- [ ] 字号足够大（手机阅读不小于14px，标题足够大）
- [ ] 每页有不同排版手法
- [ ] 封面页有强吸引力
- [ ] html2canvas导出功能正常
- [ ] 无深色背景（小红书卡片白底/浅底最佳）

### 公众号排版
- [ ] 所有样式内联（无外部CSS、无class）
- [ ] 无JavaScript
- [ ] 图片使用微信兼容格式
- [ ] 宽度适配（max-width: 100%）
- [ ] 段间距、字间距舒适

### 简历/CV
- [ ] A4单页（210×297mm）
- [ ] 打印样式 `@media print` 优化
- [ ] 无动效/无视频/无交互元素
- [ ] 信息密度高但不拥挤
- [ ] 经历倒序排列
- [ ] 联系方式清晰

### 邮件/EDM
- [ ] Table布局（不是div布局）
- [ ] 所有样式内联
- [ ] 无JavaScript
- [ ] 无外部CSS引用
- [ ] 图片有绝对路径URL
- [ ] Outlook兼容测试（无margin/padding on p/div，用table cellpadding/cellspacing）
- [ ] 有纯文本版本备选
- [ ] 退订链接（商业邮件）

## 🎯 终极三问（交付前最后问自己）

1. **辨识度**：如果去掉所有颜色只看轮廓和排版，能认出是飞鸿的设计吗？还是像一个通用模板？
2. **气质**：如果这个设计是一个人，你愿意跟他/她交朋友吗？（是否有温度、有人味、不机械）
3. **记忆点**：看完30秒后闭上眼睛，能记住哪个视觉元素？（金色线？大引言？深蓝面板？IP形象？还是什么都记不住？）
