## ⚡ 快速路径（简单修改 / 迭代）

适用场景：
- 已有页面基础上的小修改
- 单组件调整、文案替换、颜色微调
- 用户反馈后的局部修改（定位到受影响的步骤，不重新走全流程）
- 单个section添加

流程：
1. 读取现有HTML文件
2. 按照 brand-dna.md 规范直接修改
3. 对照 P0 Checklist 中与修改相关的条目快速验证
4. 如果修改影响了色彩/字体/布局 → 额外检查相关P0项
5. 交付

**不适用快速路径**：新建完整页面、超过3个section的改动、配色调整、整体风格调整。

## 📋 完整8步工作流（新建/完整设计）

---

### Step 0: 前置判断（自适应分流）

在开始之前，快速判断以下信息是否已经从用户输入中获得：

- [ ] **类型**是否明确？（属于哪个场景？核心还是Beta？）
- [ ] **受众**是否大致清晰？（给谁看）
- [ ] **内容规模**是否有概念？（几屏/几页/几张卡片/几张slides）
- [ ] **核心素材**是否已有？（文案/图片/数据/头像/IP形象）
- [ ] **硬约束**是否清楚？（必须包含什么、合作品牌色、不能用什么）

**判断规则**：
- 如果 **4项及以上** 明确 → 跳过Step 1，直接进入Step 2
- 如果 **2-3项** 明确 → 只问缺失的关键问题（不要全问）
- 如果 **0-1项** 明确 → 进入Step 1完整澄清

---

### Step 1: 澄清需求（仅在需要时）

不要机械地问5个问题。根据Step 0的判断，**只问缺失的关键问题**。

需要确认的核心问题（按需选择，不问已有答案的）：

1. **类型确认**（不确定时才问）：给出2-3个最可能的选项让用户选，不要开放式提问
2. **受众**（完全没提才问）："内容主要是给谁看的？"
3. **规模**（用户没说才问）："大概需要几屏内容/几页slides/几张卡片？"
4. **素材**（用户没提供文案才问）："有准备好的文案、图片或数据吗？"
5. **硬约束**（有特殊要求才问）："有没有必须包含的元素、合作品牌色？"

> **沟通原则**：像设计师跟客户沟通，不像填表。用自然语言，一次最多问2-3个问题。

**什么时候可以不完全用模板**：
- 模板结构完全不适合需求（但CSS变量、字体引入、基础重置必须保留）
- 用户明确要求独特的设计风格
- 但**品牌token（:root CSS变量）、字体引入、基础重置样式必须保留**

**模板修改原则**：
- 保留模板的 `:root` CSS变量
- 保留字体引入的 `<link>` 标签
- 替换内容区域，调整布局结构
- 不删除基础样式重置（`* { margin:0; padding:0; box-sizing:border-box; }` 等）

### Step 4: 设计方向定调（✨ 反AI关键步骤）

**这是最容易被跳过但最重要的步骤——AI没有"方向判断"，它直接组装。这一步决定了产出是"设计"还是"组装"。**

在填布局和组件之前，先明确以下4个方向决策：

#### 4a. 情绪基调（选1-2个主调）
- **沉稳权威**：大留白、皇家蓝主导、金色克制使用、衬线大标题、少动效
- **活力创意**：金色和酒红更活跃、非对称布局、元素溢出、动效稍多
- **艺术画册**：大图为主、文字作为视觉元素、图片溢出容器、大留白、深色面板
- **温暖人文**：Cormorant斜体更多使用、酒红占比略高、暖色调、IP形象出现
- **专业极简**：克制用色、大量留白、Inter占比更高、几何精确

#### 4b. 首屏策略（选1个）
- **视觉主导**：大图/IP形象/抽象视觉为主，文字精简
- **文字主导**：大标题+金句为主，视觉元素辅助
- **人物主导**：头像/IP形象为核心，旁边文字介绍
- **数据/成就主导**：大数字+标签为主，建立专业信任

#### 4c. 视觉签名元素（选1-2个贯穿全页）
- 金色装饰线（标题下方）
- 大装饰数字/英文（背景低透明度）
- 衬线斜体金句引言
- IP形象/头像穿插
- 金色圆点作为列表/时间线标记
- 图片溢出容器
- 深蓝面板打破节奏
- 火漆印章/花饰/角贴等书卷气元素

#### 4d. 节奏规划
- 总屏数/页数：___
- 密度节奏：密→疏→密→疏？ 还是 强→弱→强？
- 视觉惊喜点放在第几屏？
- 深色面板要不要用？放在哪里？

> **定调输出**：不需要写下来，但必须在脑中清晰。后续布局选择和组件填充都服务于这个方向。

### Step 5: 布局规划（节奏与变化）

从 `references/layouts.md` 中选取布局模式分配给各个section/页面。

**核心原则：每个section布局必须不同。布局选择服务于Step 4定的方向。**

#### 长滚动页面布局节奏策略（教程/Landing/作品集/案例/App）：
1. **第一屏（Hero）**：与Step 4b的首屏策略一致——视觉冲击或文字冲击
2. **第二屏**：与Hero形成对比（Hero大图→第二屏列表/网格；Hero文字→第二屏图文）
3. **中间sections**：交替密度——密→疏→密→疏，形成呼吸节奏
4. **视觉惊喜屏**（P1要求）：在2/3处放置——深色面板、IP形象溢出、全屏引言、大数字视觉等
5. **最后一屏**：收束——CTA、联系方式、Footer，简洁明确

#### 分页类页面布局策略（Deck/卡片/简历）：
- **演示文稿**：封面→目录→内容页（交替布局）→金句页→总结/感谢页。每页布局不同
- **图文卡片**：封面→内容页（图文混排变化）→金句页→结尾CTA。每页排版手法不同
- **简历**：单页A4，信息密度高，分区明确，不追求布局变化

**布局节奏检查**：
- [ ] 没有两个相邻section/页面使用相同布局结构
- [ ] 有疏有密，不是全程满屏内容
- [ ] 视觉惊喜点已规划（Step 4d）
- [ ] 第一屏只有一个清晰焦点（P0）
- [ ] 布局服务于Step 4定的情绪基调

### Step 6: 组件填充（品牌化，零默认样式）

从 `references/components.md` 中选取合适的组件填入各section。

**🔴 硬规则（P0级别）：禁止使用任何HTML默认样式。**

具体零默认样式要求：
- ❌ 默认 `<blockquote>`（左边框竖线是Notion/飞书千篇一律）
- ❌ 默认 `border-left` 引用块
- ❌ 无样式 `<ul>`/`<ol>`（必须自定义——金色圆点、衬线数字、自定义图标等）
- ❌ 默认 `<table>` 边框
- ❌ 默认蓝色下划线链接
- ❌ 默认按钮样式（灰色、3D效果、默认边框）
- ❌ 默认输入框样式
- ❌ 默认表单控件样式
- ✅ 所有组件从 components.md 选用
- ✅ components.md 没有的，自己设计符合brand-dna的，但绝不用浏览器默认

**品牌注入要点**（组件填充时时刻注意）：
- **链接**：皇家蓝 `#0A2463`，悬停用金色下划线/底色块
- **按钮**：主按钮皇家蓝，CTA按钮酒红，次要按钮金色描边/填充
- **引用块**：大号金色衬线引号 + Cormorant斜体 + 金色装饰线
- **列表标记**：金色圆点或 Playfair Display 衬线数字
- **标签/徽章**：金色填充（次要）、酒红填充（强调）、皇家蓝浅底（分类）
- **标题**：Playfair Display 衬线，下方可加金色装饰短线
- **分割线**：不是简单 `border-bottom`，用金色细线+渐变淡出或金色圆点分隔
- **动效**（如有）：轻、慢、克制，`cubic-bezier(0.16, 1, 0.3, 1)`，不bounce

**书卷气签名组件推荐使用**（核心场景尤其推荐）：
- 首字下沉 Drop Cap（教程首段）
- 边注 Marginalia（教程旁注）
- 火漆印章 Wax Seal（作品集/认证感）
- 照片角贴 Photo Corner（作品集/画册感）
- 金色手写签名 Signature（文章结尾/About页）
- 花饰 Fleuron ❦（章节分隔）

### Step 7: 质量自检（逐条过Checklist）

这是最关键的一步，**不能跳过**。

对照 `references/checklist.md` + `brand-dna.md` 的质量门槛逐条检查。

#### 🔴 P0 必须全过（一条不过就必须改）

- [ ] **色值精确**：搜索HTML确认三色为 `#0A2463`、`#F4D35E`、`#D8315B`，无近似色
- [ ] **色彩比例**：皇家蓝主导~55%，金色辅助~30%，酒红点缀~15%
- [ ] **金色不做大面积背景**：仅用于强调/装饰/高亮
- [ ] **正文对比度**：≥4.5:1，浅色背景不用金色正文
- [ ] **第一屏焦点**：只有一个主要视觉焦点
- [ ] **移动端适配**：<600px布局合理，文字不重叠不溢出
- [ ] **无第四主色**：成功绿仅功能提示，终端绿仅终端场景
- [ ] **禁忌检查**：无glassmorphism/弹性动画/渐变文字/AI光效/默认引用块
- [ ] **零默认样式**：所有ul/ol/blockquote/table/button/input/链接均品牌化
- [ ] **衬线+无衬线混搭**：标题Playfair Display，正文Inter，非全站一种字体
- [ ] **响应式完整**：桌面+平板+移动适配，使用clamp()
- [ ] **section/页面布局不同**：相邻区域无重复版式
- [ ] **clamp()使用**：关键字号/间距/容器用clamp()，非硬编码px
- [ ] **调性一致**：符合Step 4定的情绪基调
- [ ] **反AI检测**：截图发社交平台不会被认为是AI做的

#### 🟡 P1 应该通过（尽量满足）

- [ ] 字体策略正确：展示衬线、界面无衬线
- [ ] 整体清澈、艺术、专业、温暖

## 🔑 关键原则（时刻记住）

1. **8步不跳步** — Step 4定调是反AI的关键，不能跳过
2. **从模板开始改，不从零写** — 模板已内置品牌变量
3. **核心场景优先** — 4个Stable场景是主力，Beta场景补充
4. **每个section布局必须不同** — 避免单调重复
5. **做完必须跑Checklist** — P0全过才能交付
6. **品牌三色精确到HEX** — `#0A2463` / `#F4D35E` / `#D8315B`，不是"蓝色""黄色""红色"
7. **衬线+无衬线必须混搭** — 全站Inter = AI模板标志
8. **clamp()优先** — 字号、间距、容器尺寸用流体尺寸
9. **装饰少而准** — 金色线、大数字、引号等每页2-3处，不堆砌
10. **暖色调底色** — `#FDFBF6`暖米白，不是冷白`#FFFFFF`
11. **径向渐变制造层次** — 不用纯平色，微妙渐变增加质感
12. **书卷气增加人文温度** — 首字下沉、火漆印章、花饰等签名组件恰当使用

## 🚫 核心底线

严格遵守 `brand-dna.md` 禁忌清单，核心底线：

**截图发到任何社交平台，不会让人第一反应是"又是AI做的"。**

反AI具体策略：
- 不用对称居中卡片网格（AI最爱）
- 不用Glassmorphism/渐变文字/发光光效（AI标配）
- 字号对比极端化（大→超大，小→真小）
- 至少一个"人工判断"决策：非对称布局、元素溢出、不规则节奏、刻意留白
- Cormorant Garamond斜体引言增加人文温度
- 金色装饰线增加手工签名感
- 圆角分级（不统一大小）
- 阴影有蓝调（不通用黑色阴影）
- 留白有呼吸感（非机械等距）
- Step 4定调步骤必须执行——AI没有"调性判断"，直接组装
